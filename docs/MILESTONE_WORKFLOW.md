# Milestone Workflow

This roadmap is aligned to the course milestones from Week 6 to Week 10.

## Week 6 - Team, Scenario, Repo
Deliver:
- team roster
- one-page proposal
- GitHub repository

Required repo state:
- Git initialized
- skeleton committed
- scenario locked as `Option C`
- ownership document created

## Week 7 - Core Contracts and First Green CI
Deliver:
- repo link
- first green CI

Required repo state:
- governance token draft
- asset token draft
- vault draft
- AMM draft
- basic CI workflow
- initial tests passing

## Week 8 - Primitive Complete, 50% Coverage
Deliver:
- mid-project review checkpoint

Required repo state:
- AMM or vault flow substantially complete
- access control model in place
- `forge coverage` improving toward target
- architecture doc draft started

## Week 9 - Governance, Oracle, L2
Deliver:
- testnet addresses
- subgraph live

Required repo state:
- Governor + Timelock live in code
- Chainlink integration live
- deployment scripts working
- L2 deployment completed
- frontend reading protocol state

## Week 10 - Final Submission
Deliver:
- all code
- all docs
- demo video
- final slide deck

Required repo state:
- CI fully green
- coverage report committed
- Slither clean for High/Medium
- audit report complete
- architecture report complete
- gas report complete
- README complete

## Push and PR Discipline
- No direct coding on `main` after the initial scaffold.
- Use short-lived feature branches.
- Every feature branch ends in a PR, even if you merge it yourself.
- Never let `main` go red.
