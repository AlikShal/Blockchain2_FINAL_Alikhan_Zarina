# Presentation and Report Plan

## Report Structure
1. Executive summary
2. Scenario and protocol scope
3. Architecture diagrams
4. Contract-by-contract design
5. Governance and trust assumptions
6. Oracle design and stale-price defense
7. Security findings and fixes
8. Test strategy
9. Gas and L2 comparison
10. Deployment and verification evidence

## 15-Minute Presentation Flow
1. Problem and scenario selection
2. Protocol architecture
3. Smart contract modules
4. Governance flow
5. Oracle and security model
6. Frontend and subgraph demo
7. L2 deployment proof
8. Testing and audit evidence

## Live Demo Flow
1. Show GitHub repo and CI status
2. Run `forge test`
3. Start `anvil`
4. Run local deployment script
5. Open frontend
6. Deposit / swap / vote
7. Show indexed proposal or vault data from The Graph
8. Show verified L2 addresses as backup evidence
